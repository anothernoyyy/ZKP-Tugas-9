# 🛡️ ZKP Blockchain Authentication - TUGAS 9 PBP

> Implementasi Blockchain pada Sistem Register & Login menggunakan Zero-Knowledge Proof (zk-SNARKs) dengan Circom & SnarkJS.

## 📌 Tentang Proyek

Proyek ini mengimplementasikan sistem autentikasi **Register & Login** berbasis **blockchain** menggunakan konsensus **Zero-Knowledge Proof (ZKP)** murni. Password pengguna **tidak pernah** dikirim ke blockchain — yang tersimpan hanya **commitment** (Poseidon hash). Login diverifikasi melalui **zk-SNARK proof** (Groth16) yang divalidasi oleh **smart contract** di Ethereum Sepolia testnet.

## 🚀 Cara Menjalankan

3. **MetaMask** browser extension: https://metamask.io
4. **Sepolia ETH** (testnet): https://sepoliafaucet.com

### Step 1: Install Dependencies

```bash
npm install
```

### Step 2: Install Circom

```bash

# Windows: Download dari https://rustup.rs

# Clone & build circom
git clone https://github.com/iden3/circom.git
cd circom
cargo build --release
cargo install --path circom
cd ..

# Verify
circom --help
```

### Step 3: Install SnarkJS (global)

```bash
npm install -g snarkjs
```

### Step 4: Compile Circuit & Generate Artifacts

```bash
npm run setup
# atau

```

- Run Powers of Tau ceremony
- Generate proving key & verification key
- Export Solidity verifier (`Verifier.sol`)

## 🔐 Sirkuit ZKP (`auth.circom`)

```circom
pragma circom 2.1.6;
include "node_modules/circomlib/circuits/poseidon.circom";

template AuthVerify() {
    signal input secret;      // Private: password (field element)
    signal output commitment; // Public: Poseidon(secret)
    
    component hasher = Poseidon(1);
    hasher.inputs[0] <== secret;
    commitment <== hasher.out;
}

component main = AuthVerify();
```

### Penjelasan Logika Matematis

1. **Poseidon Hash**: Hash function yang efisien untuk circuit ZKP (ZK-friendly)
2. **Field Element**: Password di-convert ke elemen finite field BN128
3. **Constraint**: `commitment = Poseidon(secret)` — membuktikan bahwa prover mengetahui preimage dari hash
4. **Zero-Knowledge**: Verifier hanya mengetahui bahwa proof valid, TANPA mengetahui `secret` (password)

## 📜 Smart Contract

### Verifier.sol
- Generated otomatis oleh `snarkjs zkey export solidityverifier`
- Mengimplementasikan Groth16 pairing verification
- Fungsi: `verifyProof(a, b, c, pubSignals) → bool`

### ZKPAuth.sol
- Contract utama untuk autentikasi
- `register(username, commitment)` → simpan Poseidon hash
- `login(username, proof)` → verifikasi ZK proof on-chain
- Events: `UserRegistered`, `UserLoggedIn`
